D: String manipulation

﻿Given an input string and a desired output string, your task is to transform the input into the output in as few operations as possible. The two operations available to you are move and shift.

  1. move: You can move one character forward or backward 1 step through the alphabet (wrapping at the ends). ex: Given abc
    • The character a can be moved, yielding bbc or zbc
    • The character b can be moved, yielding aac or acc
    • The character c can be moved, yielding abb or abd
  2. shift: The entire string is shifted 1 step left or right (wrapping at the ends). ex: Given xyz
    • Shifted right, the string becomes zxy
    • Shifted left, the string becomes yzx

Input:

  1. The first line gives T(1≦T≦100), the number of test cases.
  2. T test cases follow.
  3. The first line of each test case gives A, the input string.
  4. The next line of each test case gives B, the desired output string.
  5. 1 ≦ |A| = |B| ≦ 500
  6. Each character is a lowercase english letter, and A is guaranteed to be transformable to B, using the available operations.

Output:

For each test case, emit a line containing the number of steps to transform A into B, followed by a new line.

Sample input:

2
abcdefg
bcdefga
aa
aa

Sample output:

1
0

Clarifications for D: String manipulation

Question: Is the definition of the "move" operation correct?
Answer: The wording was unclear and referred to moving "left and right" when it should have said "forward and backward". The problem statement has been updated to correct this.
